# MediciMonitor — Daily Log 2026-03-21

## Session Summary

### Request handled
- Ran a project review focused on the new System Monitor capabilities.
- Identified weak points in monitor reliability, alerting, and test coverage.
- Implemented focused fixes for the critical monitor issues.
- Added basic smoke/UI coverage for the new monitor endpoints and dashboard tab.

### Findings from the review
- `SystemMonitorService.RunFullScan()` executed many DB checks concurrently on a single `SqlConnection`, creating a real risk of `DataReader` / connection contention.
- Internal monitor check failures were silently swallowed, which could produce a misleadingly clean report.
- Telegram cooldown in `SystemMonitorBackgroundService` was global, so one critical alert could suppress unrelated new alerts.
- Existing smoke coverage did not validate the new `/api/monitor/*` endpoints.
- Existing dashboard Playwright coverage did not include the `sysmonitor` tab.

### Code changes made

#### 1. Monitor DB scan reliability
- Updated `Services/SystemMonitorService.cs`.
- Reworked `RunFullScan()` so DB-backed checks run through isolated `SafeRun(report, checkName, conn => ...)` calls.
- Each DB check now uses its own connection lifecycle instead of sharing one connection across concurrent tasks.

#### 2. Visible monitor failures
- Added explicit failure recording in `Services/SystemMonitorService.cs`.
- New `RecordCheckFailure(...)` writes:
  - `report.Results[checkName] = { status = "ERROR", error = ... }`
  - a `CRITICAL` monitor alert for the failed check
  - a structured warning log entry

#### 3. Alert cooldown granularity
- Updated `Services/SystemMonitorBackgroundService.cs`.
- Replaced the single global Telegram cooldown timestamp with a per-alert-key dictionary.
- Cooldown is now keyed by `Component|Message`, so unrelated incidents do not suppress one another.

#### 4. Smoke coverage for monitor endpoints
- Updated `scripts/monitor-smoke-test.sh`.
- Added checks for:
  - `/api/monitor/status`
  - `/api/monitor/history?last=1`
  - `/api/monitor/trend?hours=24`

#### 5. Dashboard coverage for System Monitor
- Updated `tests/test_dashboard.py`.
- Added `/api/monitor/status` to API endpoint validation.
- Added the `sysmonitor` tab to navigation validation.
- Added `test_system_monitor_tab()` to validate the new dashboard tab and scan button.
